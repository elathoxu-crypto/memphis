## 📝 Documentation Standards

### File Naming

- UPPERCASE.md — Top-level docs
- lowercase.md — Internal/utility docs
- kebab-case.md — Guides and tutorials

### Structure

```markdown
# Title

**Version:** X.X.X
**Date:** YYYY-MM-DD
**Status:** Draft/Production/Archived

---

## Overview

Brief description.

## Table of Contents

- [Section 1](#section-1)
- [Section 2](#section-2)

---

## Section 1

Content...

## Section 2

Content...

---

**Version:** X.X.X
**Last Updated:** YYYY-MM-DD
```

### Code Examples

- Use fenced code blocks with language
- Include comments for clarity
- Show expected output
- Provide copy-paste ready examples

## 🤝 Contributing to Docs

### How to Help

1. **Fix typos** — Edit any file
2. **Add examples** — Create PR with new examples
3. **Improve clarity** — Rewrite confusing sections
4. **Translate** — Help with Polish/English versions
5. **Update screenshots** — Keep visuals current

### Style Guide

- **Tone:** Friendly, helpful, concise
- **Person:** Second person (you)
- **Tense:** Present tense
- **Voice:** Active voice
- **Formatting:** Use markdown consistently

### Review Process

1. Fork repository
2. Make changes
3. Test locally (if applicable)
4. Submit PR
5. Address feedback
6. Merge
